OK, I've got a Hackintosh, I've got a Mac OS, and then I've got this M1 MacBook Air on a Mac OS. Now I have got two sets of keyboards and mice. Is it feasible to have one keyboard and mouse and just have my mouse and keyboard active to whatever screen my mouse is on? Lightly, this is a thing we can do.

---

**Claude Code**:

Yes, this is totally feasible — it's called **software KVM** (keyboard/video/mouse sharing). Your options:

## Option 1: Universal Control (easiest)
Apple's built-in feature — works Mac-to-Mac, no software to install.
- **Caveat**: Hackintosh may or may not work depending on how well it identifies as a Mac to iCloud/Continuity. Worth trying first.
- Enable: System Settings → General → AirPlay & Handoff → "Allow Handoff" + "Cursor and Keyboard" on both machines.
- Both must be on same network, signed into same Apple ID (or trusted together).

## Option 2: Input Leap (free, open source)
Best option if Universal Control doesn't work on the Hackintosh.
- One machine = **server**, other = **client**
- Mouse slides off the edge of one screen onto the other
- Clipboard sharing included
- Install via Homebrew: `brew install --cask input-leap`

## Option 3: Barrier (also free)
Older fork of Synergy, same concept as Input Leap. Input Leap is the more actively maintained successor.

## Recommendation
1. Try **Universal Control** first — zero setup if it works
2. Fall back to **Input Leap** if the Hackintosh isn't recognized

The M1 MacBook Air should be the **server** (more reliable as host), Hackintosh as the **client**.

**Claude Code**:

That rules out Universal Control — it requires the same Apple ID across devices.

**Input Leap** is your answer then. Since both machines run macOS it's straightforward:

1. Install on both: `brew install --cask input-leap`
2. Set the **Hackintosh** (your main account) as the **server**
3. Set the **MacBook Air** as the **client**
4. In the server config, add the MacBook's screen and position it left/right of the Hackintosh screen
5. Mouse slides off the edge → control transfers, keyboard follows

No Apple ID needed — it's purely network-based over your local WiFi/LAN.

**Human**:

okay, give me step-by-step instructions. I believe I've installed it on both. What do I do next

---

**Claude Code**:

**On the Hackintosh (Server):**

1. Open Input Leap
2. Select **"Server"**
3. Click **"Configure Server..."**
4. You'll see a grid — drag a new screen tile next to the existing one (position it to match where the MacBook physically sits — left or right)
5. Double-click the new screen tile → set the name to the **MacBook's hostname** (find it on the Mac: System Settings → General → About → Name, e.g. `MacBook-Air.local`)
6. Click OK, then **Start**
7. Note the Hackintosh's **IP address** (System Settings → Network, or `ipconfig getifaddr en0` in Terminal)

---

**On the MacBook Air (Client):**

1. Open Input Leap
2. Select **"Client"**
3. In the server address field, enter the **Hackintosh's IP address**
4. Click **Start**

---

**Grant permissions (macOS will prompt):**

On both machines, go to:
- System Settings → Privacy & Security → **Accessibility** → add/enable Input Leap
- System Settings → Privacy & Security → **Input Monitoring** → add/enable Input Leap

Then restart Input Leap on both if needed.
