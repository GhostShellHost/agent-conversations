## APFEL (Arthur-Ficial/apfel)

**Apple Intelligence From the command linE** - A CLI and HTTP server that wraps Apple's built-in on-device LLM (FoundationModels framework) on Apple Silicon Macs.

### What It Does

Every M-series Mac has a **free, on-device LLM** built in via Apple Intelligence, but Apple only exposes it through Siri. APFEL unlocks it:

- **CLI**: `echo "summarize this" | apfel` — pipe-friendly, file attachments, JSON output
- **OpenAI-compatible server**: `apfel --serve` — drop-in replacement at `localhost:11434`, works with any OpenAI SDK
- **Tool calling**: Full function-calling support with schema conversion
- **100% on-device**: No API keys, no cloud, no cost, 4096-token context

### Tech Stack
- Swift 6.3, FoundationModels framework (macOS 26+)
- 1.9k stars, 60 forks, 163 commits, actively maintained
- Install: `brew install Arthur-Ficial/tap/apfel`

## What Runs Where

### TTS (`joule-tts` / `say` command)
- **Uses:** System speech synthesis (AVSpeechSynthesizer)
- **Neural Engine:** Yes, for neural voices like Samantha
- **RAM:** Minimal (~50-100MB when active)
- **CPU:** Very low usage
- **Offline:** Yes, fully on-device

### APFEL (Apple Foundation Model)
- **Uses:** FoundationModels framework
- **Neural Engine:** Yes, for inference acceleration
- **RAM:** ~2-4GB when actively running (model loaded in memory)
- **CPU:** Spikes during inference, idle otherwise
- **Offline:** Yes, 100% on-device

---

## Key Differences vs Cloud Models

| Aspect | Cloud (Kimi, Claude, etc.) | APFEL (On-Device) |
|--------|---------------------------|-------------------|
| Your RAM | ~0 (just API calls) | 2-4GB during inference |
| Your CPU | ~0 | Uses Neural Engine |
| Tokens | Cost money / rate limited | Free, unlimited |
| Privacy | Data leaves Mac | Never leaves Mac |
| Works offline | No | Yes |
| Speed | Network latency | Instant (local) |

---

## The Trade-off

**APFEL is NOT zero-memory** — it does load into RAM when running. But:

1. **launchd keeps it running** as a background service (~2-4GB resident)
2. **Neural Engine handles inference** efficiently (low power, fast)
3. **No token costs** — unlimited local queries
4. **Always available** — even when internet is down

**TTS is essentially free** — negligible RAM, uses system frameworks that are always loaded anyway.

---

## Bottom Line

- **TTS:** Virtually zero overhead (system-level, always available)
- **APFEL:** Uses 2-4GB RAM but gives you free, private, offline inference

For your 8GB MacBook Air, APFEL running means ~25-50% of RAM used. That is the trade-off for having a local fallback that works when cloud APIs fail.
